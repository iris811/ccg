Use Claude Code Where You Work

Terminal, IDE, Desktop, Web & iOS, Slack — 5가지 환경에서 사용하는 방법

1

Terminal

가장 강력한 풀 기능 CLI — 파일 편집, 명령 실행, 프로젝트 전체 관리

▼
Image source: claude.com/product/claude-code

Step 1 — 설치

운영체제에 맞는 방법으로 설치합니다:

# macOS / Linux / WSL (권장)
curl -fsSL https://claude.ai/install.sh | bash

# Windows PowerShell (권장)
irm https://claude.ai/install.ps1 | iex

# Windows CMD
curl -fsSL https://claude.ai/install.cmd -o install.cmd && install.cmd && del install.cmd

# Homebrew (macOS)
brew install --cask claude-code

# WinGet (Windows)
winget install Anthropic.ClaudeCode
Windows 사용자: 설치 전 Git for Windows가 먼저 설치되어 있어야 합니다.

[Option] 설치 확인

설치 후 아래 명령어로 설치 유형과 버전을 확인합니다:

claude doctor

Step 2 — 로그인

첫 실행 시 자동으로 로그인 화면이 나타납니다. Claude 구독(Pro, Max, Team, Enterprise) 또는 Anthropic Console 계정이 필요합니다.

claude

Step 3 — 프로젝트에서 시작

# 프로젝트 폴더로 이동 후 실행
cd your-project
claude

# 특정 모델로 시작
claude --model opus

# 이전 세션 이어하기
claude --continue

# 권한 확인 없이 자동 실행 (bypass 모드) — CI/자동화 환경에서 유용
claude --dangerously-skip-permissions

# Chrome 브라우저 제어 모드로 시작 (Beta, Chrome 확장 프로그램 필요)
claude --chrome
주의: --dangerously-skip-permissions는 파일 수정·명령 실행 등 모든 작업을 승인 없이 자동으로 처리합니다. 신뢰할 수 있는 환경에서만 사용하세요.

Step 4 — 프로젝트 초기화 (권장)

# CLAUDE.md 생성 — 프로젝트 규칙/아키텍처/스타일 정의
/init

Step 5 — 스크립트 & CI용 Non-Interactive 모드

# 한 번 실행하고 결과 출력
claude -p "이 코드에서 버그를 찾아줘"

# 파이프로 입력 전달
cat error.log | claude -p "오류 분석해줘"

# 로그 모니터링 → Slack 알림
tail -f app.log | claude -p "이상 징후 발견 시 Slack으로 알려줘"

# CI에서 번역 자동화
claude -p "새 문자열을 한국어로 번역하고 PR 올려줘"
요구 사항: macOS 13.0+, Windows 10 1809+, Ubuntu 20.04+. 4GB+ RAM.
자동 업데이트: Native Install (curl/PowerShell/CMD)로 설치하면 백그라운드에서 자동 업데이트됩니다. Homebrew/WinGet은 수동 업데이트가 필요합니다 (brew upgrade claude-code).
더 알아보기: 설치 중 문제가 발생하면 공식 설치 가이드를 참고하세요. Claude Code의 설치 및 활용 방법은 claude.com/product/claude-code에서도 확인할 수 있습니다.
2

IDE

VS Code, Cursor, JetBrains, Antigravity — 에디터 안에서 바로 사용

▼
Image source: claude.com/product/claude-code

VS Code / Cursor / Antigravity — 공식 확장 프로그램

  1. Ctrl+Shift+X → "Claude Code" 검색 → Install
  2. Command Palette (Ctrl+Shift+P) → "Claude Code" → Open in New Tab

VS Code 확장 주요 기능:

  • 인라인 Diff 뷰어로 코드 변경 사항을 시각적으로 리뷰
  • Plan 모드로 실행 전 계획 검토
  • @파일명으로 파일 참조 (라인 범위 지정 가능)
  • 여러 대화를 탭으로 관리
  • Cmd/Ctrl+Esc로 에디터와 Claude 간 포커스 전환

JetBrains (IntelliJ, PyCharm, WebStorm 등) — Beta 플러그인

  1. Settings → Plugins → Marketplace에서 "Claude Code" 검색 → Install
  2. IDE를 재시작합니다.

인터랙티브 diff 뷰잉과 선택 컨텍스트 공유를 지원합니다.

기타 에디터 — 터미널에서 직접 실행

  1. 터미널 열기: Ctrl+` 또는 메뉴에서 Terminal → New Terminal
  2. Claude Code 실행: 
    claude
    에디터의 내장 터미널은 이미 프로젝트 루트에 위치하므로 바로 실행하면 됩니다.
  3. 작업 시작: Claude가 에디터에 열린 파일들과 같은 프로젝트에 접근하여 코드를 읽고, 수정하고, 실행합니다. 변경 사항이 에디터 파일 탐색기에 실시간 반영됩니다.
Tip: 어떤 에디터를 쓰든 내장 터미널에서 claude만 실행하면 됩니다. VS Code/JetBrains 전용 확장은 인라인 diff, @mention 등의 추가 기능을 제공합니다. 터미널에서 claude --ide 또는 세션 내 /ide로 IDE와 자동 연결할 수도 있습니다.
3

Desktop

독립 실행형 앱 — 시각적 diff 리뷰, 멀티 세션, 스케줄링, 클라우드 세션

▼
Image source: claude.com/product/claude-code

Step 1 — 다운로드 & 설치

  • macOS (Intel & Apple Silicon): claude.ai/download에서 DMG 다운로드
  • Windows (x64): 동일 페이지에서 EXE 다운로드

Step 2 — 로그인 & Code 탭 열기

  1. 앱을 실행하고 Anthropic 계정으로 로그인합니다.
  2. Code 탭을 클릭하여 코딩 세션을 시작합니다.

Step 3 — 주요 기능 활용

  • 시각적 Diff 리뷰: 코드 변경사항을 깔끔한 UI로 확인
  • 멀티 세션: 여러 작업을 나란히 실행
  • 스케줄 태스크: 반복 작업 예약
  • 클라우드 세션: 웹에서 시작한 세션을 데스크톱에서 이어서 작업

터미널 세션을 Desktop으로 전환

CLI에서 작업하다가 시각적 리뷰가 필요할 때:

/desktop
Tip: Desktop 앱은 IDE나 터미널 없이도 독립적으로 사용할 수 있어, 비개발자 팀원이나 코드 리뷰 용도로 적합합니다. 유료 구독이 필요합니다.
4

Web & iOS

브라우저와 모바일에서 — 설치 없이 바로, 장시간 작업을 맡기고 나중에 확인

▼
Image source: claude.com/product/claude-code

Step 1 — 접속

claude.ai/code에 접속하여 로그인합니다. 별도 설치가 필요 없습니다.

Step 2 — GitHub 연결

GitHub 계정을 연결하고 작업할 리포지토리를 인증합니다. Claude가 코드를 읽고 수정하고 PR을 생성할 수 있게 됩니다.

Step 3 — 코딩 세션 시작

작업 내용을 입력하면 Claude가 자동으로 코드를 분석하고 작업을 시작합니다.

  • 장시간 작업: 작업을 맡기고 나중에 돌아와 결과를 확인할 수 있습니다.
  • 로컬에 없는 리포: 로컬에 클론하지 않은 리포지토리도 작업 가능합니다.
  • 병렬 실행: 여러 작업을 동시에 실행할 수 있습니다.

iOS 앱에서도 동일하게

Claude iOS 앱에서도 Code 기능을 사용할 수 있습니다. 이동 중에 작업을 시작하거나 확인할 수 있습니다.

환경 간 세션 이동

# 웹 세션을 로컬 터미널로 가져오기
claude --teleport

# 로컬 세션을 웹/폰에서 접속 가능하게 만들기
/remote-control

# CLI에서 웹 세션 생성
claude --remote "로그인 버그 수정해줘"
Tip: 세션은 특정 환경에 고정되지 않습니다. /teleport로 웹→터미널, /desktop으로 터미널→데스크톱 앱, /remote-control로 터미널→웹/폰으로 자유롭게 이동할 수 있습니다.
5

Slack

팀 채팅에서 @Claude 멘션으로 코딩 작업을 바로 시작

▼
Image source: claude.com/product/claude-code

Step 1 — Claude 앱 설치

  1. Slack 워크스페이스 관리자가 Slack App Marketplace에서 Claude 앱을 설치합니다.
  2. 설치 후 Slack의 Apps 섹션에서 "Claude"를 클릭 → App Home 탭 → Connect를 눌러 Claude 계정을 연결합니다.

Step 2 — GitHub 연결 확인

claude.ai/code에서 GitHub 계정이 연결되어 있고, 작업할 리포지토리가 인증되어 있는지 확인합니다.

Step 3 — 라우팅 모드 설정

Claude App Home에서 메시지 처리 방식을 설정합니다:

모드동작
Code only모든 @멘션을 Claude Code 세션으로 라우팅. 개발 전용 팀에 적합.
Code + Chat메시지를 분석하여 코딩 작업은 Code로, 일반 질문은 Chat으로 자동 라우팅.

Step 4 — 채널에 초대 & 사용

# 채널에 Claude 초대
/invite @Claude

# 코딩 작업 요청
@Claude 로그인 페이지의 인증 버그를 조사하고 수정해줘

작동 방식

  1. @Claude 멘션으로 코딩 작업을 요청합니다.
  2. Claude가 자동으로 코딩 의도를 감지하고 claude.ai/code에 세션을 생성합니다.
  3. Slack 스레드에 진행 상황이 업데이트됩니다.
  4. 완료되면 요약과 함께 View Session, Create PR 버튼이 표시됩니다.
Tip: 스레드에서 @멘션하면 전체 스레드 컨텍스트를 참고합니다. 버그 리포트 스레드에서 바로 @Claude를 멘션하면 팀 논의 내용까지 이해한 채로 작업합니다.
요구 사항: Pro/Max/Teams/Enterprise 플랜 + Claude Code on the web 접근 권한 + GitHub 연결. 현재 GitHub 리포지토리만 지원합니다.

⚠ 권한 & 보안 플래그

주의: 아래 플래그들은 보안에 영향을 줍니다. 신뢰할 수 있는 환경에서만 사용하세요.
Flag설명예시
--dangerously-skip-permissions 모든 권한 프롬프트를 건너뜀. 파일 수정, 명령어 실행 등을 확인 없이 자동 허용. CI/CD나 자동화 스크립트에서 주로 사용. 반드시 --allowedTools와 함께 사용 권장. claude --dangerously-skip-permissions
--permission-mode plan Plan 모드. 읽기 전용으로 동작. 코드를 수정하지 않고 분석/계획만 수행. claude --permission-mode plan
--permission-mode auto-accept 자동 승인 모드. 대부분의 도구 사용을 자동 허용. 신뢰할 수 있는 프로젝트에서 사용. claude --permission-mode auto-accept
--allowedTools <rules> 특정 도구만 프롬프트 없이 허용. 패턴 매칭 지원. claude --allowedTools "Bash(git *)" "Read"
--disallowedTools <tools> 특정 도구 사용을 완전히 차단. claude --disallowedTools "Bash(rm *)"
--sandbox 샌드박스 환경에서 실행. 파일시스템과 네트워크를 격리하여 안전하게 작업. claude --sandbox
실전 활용 예시:
# CI/CD 파이프라인에서 자동 코드 리뷰
claude --dangerously-skip-permissions \
  -p "이 PR의 코드를 리뷰하고 문제점을 알려줘"

# 안전한 자동화: 특정 명령만 허용
claude --dangerously-skip-permissions \
  --allowedTools "Read" "Grep" "Bash(git log *)" \
  -p "최근 커밋을 분석해줘"

# Plan 모드로 안전하게 분석만
claude --permission-mode plan \
  -p "이 프로젝트의 아키텍처를 분석해줘"

인기 Skills

직접 만드는 커스텀 스킬과 skills.sh의 인기 커뮤니티 스킬

커스텀 Skill 만들기

자신만의 스킬을 만들어 반복 작업을 자동화할 수 있습니다:

# ~/.claude/skills/my-review/SKILL.md 생성

---
name: my-review
description: 팀 코딩 컨벤션에 맞춰 코드 리뷰
user-invocable: true
allowed-tools: Read, Grep, Bash
---

코드를 리뷰할 때 다음 규칙을 확인해주세요:
1. 함수명은 camelCase
2. 컴포넌트는 PascalCase
3. 모든 export에 JSDoc 주석 필수
4. console.log 제거 확인

위치: 개인용 ~/.claude/skills/ · 프로젝트용 .claude/skills/

활용 팁 & Best Practices

Claude Code를 더 효과적으로 사용하기 위한 실전 팁

✍ CLAUDE.md 적극 활용

프로젝트 루트에 CLAUDE.md를 만들어 아키텍처, 코딩 규칙, 기술 스택을 정의하세요. Claude가 프로젝트 맥락을 이해하고 더 정확한 코드를 작성합니다.

🛠 Plan 모드로 먼저 설계

큰 작업 전에 --permission-mode plan으로 시작하세요. Claude가 코드를 수정하지 않고 계획만 세워서 검토 후 실행할 수 있습니다.

🚀 Worktree로 병렬 작업

claude -w feature-name으로 격리된 환경에서 여러 작업을 동시에 진행하세요. 서로 충돌 없이 독립적으로 작업됩니다.

💰 비용 관리

/context로 사용량을 확인하고, 간단한 작업은 /fast on으로 전환하세요. /compact로 컨텍스트를 정리하면 토큰을 절약합니다.

💡 ultrathink 활용

복잡한 알고리즘 설계나 아키텍처 결정 시 프롬프트에 "ultrathink"를 포함하면 Claude가 더 깊이 사고합니다.

🔌 세션 관리

/rename으로 세션에 이름을 붙이고, claude --resume name으로 나중에 이어서 작업하세요. 긴 작업도 끊김 없이 진행됩니다.

자주 사용하는 워크플로우

# 1. 프로젝트 시작
cd my-project
claude /init              # CLAUDE.md 생성
claude                    # 세션 시작

# 2. 복잡한 작업 계획 후 실행
claude --permission-mode plan
# (계획 검토 후 실행 모드로 전환)

# 3. 코드 리뷰
git diff | claude -p "이 변경사항을 리뷰해줘"

# 4. 디버깅
cat error.log | claude -p "이 에러의 원인을 분석해줘"

# 5. 반복 모니터링
/loop 5m 배포 상태를 확인하고 에러가 있으면 알려줘

# 6. 원격 접속
/remote-control           # QR 코드 생성, 폰에서 접속

# 7. 세션 관리
/rename auth-refactor     # 세션 이름 지정
/context                  # 사용량 확인
/compact                  # 컨텍스트 정리
# 나중에 이어서 작업
claude --resume auth-refactor
